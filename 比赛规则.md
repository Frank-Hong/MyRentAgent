## 评分规则
|类型 |常量 |积分 |说明 |例子|
   |---|---|---|---|---|
|聊天类 |Chat |5 分 |基础聊天 |1. 基本对话能力|
|单轮对话简 单任务 |Single |10-15 分 |单条件简单查询 多条件复杂查询|1. 查询两居室房源 2. 简单查询朝阳区房源 3.海淀+两居+精装修+近地铁+电梯+民水民电+预算 10000+60平以上|
|多轮对话复杂任务 |Multi |20-30 分|1. 多轮简单、复杂查询 2. 多房源对比 3. 边聊天边查询并租房|1. 从吐槽当前房子到找新房子再到租房（5 轮）2. 链家/安居客/58 同城等多家交叉筛选→比价→决策（6 轮）3. 从电梯→地铁→装修→朝向→采光→入住反复筛选（7 轮）|



**评测流程**：判题系统会按照用例顺序调用选手 Agent 接口执行评测用例并获得选手
Agent 对用例的最终执行结果，判题系统会根据结果给出当前用例得分，每次评测资
源上限为 300 时间片（该时间片只是抽象的一个数据，非真实的时间），300 时间片内
所有跑完用例数总分为最终分数，同时计算本轮评测消耗总 token 数。

**时间片消耗计算规则**：判题系统会按照模型调用次数消耗的时间片来算：每次调用一
次模型消耗 token 数折算时间片公式为：t = 1 个时间片+max（0，（nk-1k））*m，n
为本次模型调用消耗 token 数，其中 m 为 0.3，时间片 t 计算结果为小数时向上取整

**计分规则**：用例有不同类型，基础聊天类只要返回结果即可给分（单个用例），单论对
话简单任务和多轮对话复杂任务类用例，答案可能存在 0 到多个，计分规则按照实际
命中情况给分，即给出的答案与用例答案匹配的数量越多，当前用例得分越高。

**排名规则**：首先根据总得分计算排名，得分一致，则按照本轮运行所有用例消耗总
token 进行比较，消耗 token 越少，排名越高
